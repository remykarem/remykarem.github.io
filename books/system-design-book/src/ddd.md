First impressions: DDD seems to apply to application code more than to system architecture. Spring Boot also seems to use these terms.

* Ubiquitous Language
* Bounded Context
* Entity
* Value Objects (e.g. `Money` instead of `Decimal`)
* Aggregate
* Domain Service ("process")
* Repository
