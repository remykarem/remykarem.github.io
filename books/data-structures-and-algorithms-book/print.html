Classes of problems

Decision problem: a problem with a yes or no answer.

P

polynomial time

Class of problems that can be solved by an algorithm in polynomial time

NP

nondeterministic polynomial time

If I have unlimited compute power, I am capable of solving any problem in at most polynomial time.

Their solutions can be verified within polynomial time. ‚ÄúEasy to check but hard to crack.‚Äù

  • NP-complete problems

    A subset of problems X in NP for which it is possible to reduce any other NP problem Y to X in polynomial time.

    i.e. if we know how to solve X quickly, then we can solve Y.

    A class of computational problems where an effifecint algorithm solution has yet to be found

NP-hard

There exists no polynomial time algorithm to verify a solution to such problems. Eg. chess

üí° A hash map in Python is a `dict`. 
class Hashmap:

    def __init__(self):
        self.table = [None] * 10

    def put(self, key, value):
        idx = hash(key) % 10
        if self.table[idx] is None:
            self.table[idx] = [(key, value)]
        else:
            for i, el in enumerate(self.table[idx]):
                k,v = el
                if k == key:
                    self.table[idx][0] = (key,value)
                    return
            self.table[idx].append((key,value))

    def get(self, key):
        idx = hash(key) % 10
        for k,v in self.table[idx]:
            if k==key:
                return v
        raise

Tree

  • Extension of linked list

Tree characteristics:

  • No. of children: m-ary tree is one where each node must have a maximum of m children
  • Balanced
  • Leaf-similar
  • Diameter ‚Äî length of longest path between any two nodes

Node characteristics:

  • Type ‚Äî one of root, internal, leaf. Or parent vs. child
  • Depth or level
  • Tilt
  • Cousins ‚Äî two nodes are cousins if they have the same depth but different parents
  • All nodes must be connected. There must not be any cycles.

Binary tree

Every parent node has at most 2 children.

üí° A **complete binary tree**, all levels are full (except maybe the deepest, where the nodes are as far to the left as possible).

Maximum binary tree


  4 6 3 10000 5 1

Given an array,

  1. parent ‚Äî maximum of the array
  2. left subtree ‚Äî the array to the left of the maximum
  3. right subtree ‚Äî the array to the right of the maximum

AVL tree

An AVL tree is a height-balanced BST.

Height-balanced means the heights of the left and right subtrees differ by at most one.

Why need to balance height?

To reduce the time for traversal and insertion.

From array

To construct an AVL tree:

  1. Sort the array in ascending order.
  2. Take the middle element as the root.
  3. The elements to the left are for the left subtree, right are for right subtree.

Shortest path algorithm

At every node, visit the shortest path to the next node (read: greedy) until you reach the end node.

Bubble sort

  • Average: O(n2)
  • Best: O(n) (sorted)
  • Worst: O(n2)
  • Space: O(1), in-place

Merge sort

  • O(nlogn)
  • Auxiliary space: O(n)

Quick sort

  • Worst case: O((n-1)) = O(n)

Path finding

Heuristic function: estimated cost path from current node to goal

Admissible heuristic function is a heuristic function that does not overestimate the actual cost to get to goal (i.e. heuristic cost is more than the lowest possible cost)

Recursion

Making recursive calls is like making use of a stack data structure.

This is because many programming languages are stack-based (citation needed).

How to think about recursion?

Managing states

Global states can be managed via side effects.

Handling of states between function calls: Local states can be

  • returned -- useful when "bubbling up"
  • passed in -- useful when following together with the traversal

Apart from passing in nodes, you can also pass other things/nodes.
